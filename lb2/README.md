## CVE-2025-23120

**CVE-2025-23120** — это критическая уязвимость удалённого выполнения кода (**RCE**) в программном обеспечении **Veeam Backup & Replication**.

### Описание уязвимости
Суть уязвимости заключается в обходе механизма безопасности, который использовал **ненадёжный чёрный список** для блокировки опасных команд.

Основная проблема состоит в том, что атакующий может **без каких-либо учётных данных** отправить на порт **9380** сервера Veeam специально сформированное имя хоста.

Для обхода встроенных фильтров используются такие техники, как:
- экранирование кавычек;
- передача вредоносного кода в формате **IPv6-адреса**.

В результате становится возможным выполнение **произвольного кода** на сервере с максимальными привилегиями **SYSTEM**, что фактически эквивалентно полному захвату контроля над системой.

### Последствия
Эксплуатация уязвимости позволяет злоумышленнику:
- полностью управлять сервером;
- похищать резервные копии данных;
- шифровать резервные копии;
- уничтожать резервные копии данных компании.

### Источник
[WatchTowr Labs — CVE-2025-23120](https://labs.watchtowr.com/by-executive-order-we-are-banning-blacklists-domain-level-rce-in-veeam-backup-replication-cve-2025-23120/)

## CVE-2025-24813

**CVE-2025-24813** — уязвимость типа *эквивалентность пути* (`file.Name`, «внутренняя точка»), которая может приводить к:
- удалённому выполнению кода (**RCE**);
- раскрытию конфиденциальной информации;
- внедрению вредоносного содержимого в загружаемые файлы  
через **сервлет по умолчанию с поддержкой записи** в **Apache Tomcat**.

### Затронутые версии Apache Tomcat
Уязвимость затрагивает следующие версии:
- **11.x**: с `11.0.0-M1` по `11.0.2`;
- **10.x**: с `10.1.0-M1` по `10.1.34`;
- **9.x**: с `9.0.0.M1` по `9.0.98`.

Также известно, что следующие устаревшие версии (на момент публикации CVE) подвержены уязвимости:
- **8.5.x**: с `8.5.0` по `8.5.100`.

Более старые устаревшие версии также могут быть уязвимы.

### Условия для раскрытия информации и внедрения содержимого
Злоумышленник мог просматривать чувствительные к безопасности файлы и/или внедрять в них содержимое, если выполнялись **все** следующие условия:
- включена запись для сервлета по умолчанию (*по умолчанию отключена*);
- включена поддержка частичного **PUT** (*по умолчанию включена*);
- целевой URL чувствительных загрузок являлся подкаталогом URL публичных загрузок;
- злоумышленнику были известны имена чувствительных загружаемых файлов;
- чувствительные файлы также загружались с использованием частичного **PUT**.

### Условия для удалённого выполнения кода (RCE)
Удалённое выполнение кода становилось возможным, если выполнялись **все** следующие условия:
- включена запись для сервлета по умолчанию (*по умолчанию отключена*);
- включена поддержка частичного **PUT** (*по умолчанию включена*);
- приложение использовало файловое хранилище сессий Tomcat с расположением по умолчанию;
- приложение включало библиотеку, которая может быть использована в атаке десериализации.

### Рекомендации
Пользователям настоятельно рекомендуется обновиться до исправленных версий:
- **11.0.3**;
- **10.1.35**;
- **9.0.99**.

[NVD — CVE-2025-24813](https://nvd.nist.gov/vuln/detail/CVE-2025-24813)

## CVE-2025-29927

**CVE-2025-29927** — критическая уязвимость в популярном фреймворке **Next.js**, позволяющая обойти механизм авторизации, реализованный с использованием **middleware**.

### Описание уязвимости
**Middleware** (функция промежуточной обработки) — это набор функций, которые последовательно обрабатывают входящие HTTP-запросы веб-приложения. Одной из ключевых задач middleware является проверка прав доступа к защищённым разделам приложения.

Для предотвращения бесконечных циклов рекурсивных запросов Next.js использует внутренний HTTP-заголовок:

Однако при самостоятельной передаче этого заголовка с корректным значением злоумышленником, фреймворк **пропускает этап обработки запроса в middleware**. В результате:
- не выполняется проверка авторизации;
- не проверяется наличие cookie или иных токенов доступа;
- пользователь получает прямой доступ к защищённым разделам веб-приложения.

### Затронутые версии
Уязвимости подвержены **все версии Next.js** в диапазоне:
- с **11.1.4** по **15.2.2** включительно.

### Последствия
Эксплуатация уязвимости может привести к:
- обходу механизмов аутентификации и авторизации;
- несанкционированному доступу к защищённым ресурсам;
- компрометации пользовательских данных и бизнес-логики приложения.

### Рекомендации
Для устранения уязвимости рекомендуется:
- **обновить Next.js до актуальной исправленной версии**;
- дополнительно проверить логику авторизации на стороне backend-API.

### Источник
[NVD — CVE-2025-29927](https://nvd.nist.gov/vuln/detail/CVE-2025-29927)

## CVE-2025-22224

**CVE-2025-22224** — критическая уязвимость типа *состояние гонки* (**TOCTOU — Time Of Check / Time Of Use**), связанная с ошибками синхронизации при работе с общими ресурсами.

### Описание уязвимости
Уязвимость присутствует в гипервизорах:
- **VMware ESXi**;
- **VMware Workstation**.

Ошибка синхронизации приводит к **записи за пределами допустимых границ памяти** (*out-of-bounds write*).

Эксплуатация уязвимости позволяет злоумышленнику, обладающему правами **локального администратора внутри виртуальной машины**, выполнить **произвольный код** в контексте процесса **VMX**, запущенного на хост-узле.

### Последствия
В результате успешной эксплуатации возможно:
- выполнение произвольного кода на уровне процесса гипервизора;
- выход из виртуальной машины на хостовую систему (*VM escape*);
- полная компрометация узла виртуализации.

Отмечается, что уязвимость **уже используется в реальных атаках**, что значительно повышает уровень риска.

### Условия эксплуатации
- наличие прав локального администратора в виртуальной машине;
- доступ к уязвимому гипервизору VMware.

### Рекомендации
Рекомендуется:
- незамедлительно установить обновления безопасности VMware;
- ограничить права пользователей внутри виртуальных машин;
- усилить мониторинг активности VMX-процессов.

### Источник
[Positive Technologies — CVE-2025-22224](https://ptsecurity.com/research/trending-vulnerabilities/CVE-2025-22224/)

## CVE-2024-48990

**CVE-2024-48990** — уязвимость локальной эскалации привилегий (**LPE, Local Privilege Escalation**) в утилите **needrestart**, обнаруженная в бинарном файле.

### Описание уязвимости
19 ноября 2024 года компания **Qualys** публично раскрыла информацию о пяти уязвимостях в утилите *needrestart*. Данные уязвимости позволяют локальному злоумышленнику повысить свои привилегии в системе.

Уязвимость **CVE-2024-48990** затрагивает версии **needrestart до 3.8** включительно. В дистрибутиве **Ubuntu** утилита применяется по умолчанию, начиная с версии **21.04**. Также проблема актуальна для:
- **Debian**;
- **Fedora**;
- других дистрибутивов Linux.

### Что такое LPE
**LPE (Local Privilege Escalation)** — это класс уязвимостей, позволяющих злоумышленнику повысить уровень своих привилегий в системе и получить доступ к операциям, доступным только администраторам (например, **root-доступ**).

### Последствия
Эксплуатация уязвимости может привести к:
- получению полного контроля над системой;
- установке вредоносного программного обеспечения;
- обходу механизмов безопасности;
- несанкционированному доступу к конфиденциальным данным;
- управлению системой от имени администратора.

### Рекомендации
Рекомендуется:
- обновить утилиту **needrestart** до исправленной версии;
- регулярно устанавливать обновления безопасности ОС;
- ограничить доступ локальных пользователей к системе.

### Источник
[Habr — CVE-2024-48990](https://habr.com/ru/articles/866582/)

## CVE-2024-1086

**CVE-2024-1086** — уязвимость типа *использование памяти после освобождения* (**Use-After-Free**) в компоненте **netfilter: nf_tables** ядра **Linux**, которая может быть использована для **локального повышения привилегий (LPE)**.

### Описание уязвимости
Уязвимость связана с некорректной обработкой кодов ошибок в функции:
Данная функция допускает положительные значения в качестве ошибки сброса в вердикте хука. В результате функция:

может вызвать ситуацию **двойного освобождения памяти** (*double free*) при обработке вердикта **NF_DROP** с ошибкой сброса, логически схожей с **NF_ACCEPT**.

### Механизм эксплуатации
Ошибка управления памятью приводит к состоянию *use-after-free*, которое может быть использовано локальным злоумышленником для:
- повреждения структуры данных ядра;
- выполнения произвольного кода в контексте ядра;
- повышения привилегий до уровня **root**.

### Последствия
Эксплуатация уязвимости может привести к:
- локальной эскалации привилегий;
- компрометации ядра Linux;
- полному контролю над системой.

### Условия эксплуатации
- наличие локального доступа к системе;
- возможность взаимодействия с подсистемой **nftables** / **netfilter**.

### Рекомендации
Рекомендуется:
- обновить ядро Linux до версии с исправлением уязвимости;
- ограничить доступ к управлению **nftables** для непривилегированных пользователей;
- применять механизмы дополнительной изоляции (например, namespaces, SELinux).

### Источник
[GitHub — CVE-2024-1086](https://github.com/Notselwyn/CVE-2024-1086?tab=readme-ov-file)

## CVE-2024-21338

**CVE-2024-21338** — уязвимость ядра **Windows**, приводящая к **повышению привилегий (LPE)**.

### Описание уязвимости
Уязвимость находится в диспетчере **IOCTL** (управление вводом/выводом) драйвера **appid.sys**, который является центральным компонентом **AppLocker** — технологии белого списка приложений, встроенной в Windows.

Уязвимый управляющий код:
предназначен для вычисления интеллектуального хеша исполняемого файла образа. Данный IOCTL предоставляет вызывающей стороне гибкость, позволяя указывать, каким образом драйвер должен запрашивать и считывать файл для хеширования.

### Технические детали
Гибкость реализации достигается за счёт использования **двух указателей на функции ядра**, передаваемых во входном буфере IOCTL:
- указатель функции обратного вызова для запроса размера хешируемого файла;
- указатель функции обратного вызова для чтения данных, подлежащих хешированию.

Однако уязвимый IOCTL **оставался доступным из пользовательского пространства**, что позволило злоумышленнику:
- передавать произвольные указатели функций ядра;
- частично контролировать данные, передаваемые в качестве первого аргумента вызываемой функции.

### Вектор атаки
Злоумышленник из пользовательского пространства может:
- обмануть ядро Windows;
- заставить его вызвать **произвольную функцию ядра**;
- получить высокий уровень контроля над передаваемыми параметрами.

Такой сценарий представляет собой **идеальные условия для эксплуатации**, позволяя добиться повышения привилегий до уровня ядра.

### Последствия
Эксплуатация уязвимости может привести к:
- выполнению произвольного кода в режиме ядра;
- локальной эскалации привилегий до **SYSTEM**;
- полной компрометации операционной системы.

### Рекомендации
Рекомендуется:
- установить актуальные обновления безопасности Windows;
- ограничить запуск и использование потенциально опасных драйверов;
- применять дополнительные механизмы защиты ядра (например, HVCI, Credential Guard).

### Источник
[Gen Digital — CVE-2024-21338](https://www.gendigital.com/blog/insights/research/lazarus-and-the-fudmodule-rootkit-beyond-byovd-with-an-admin-to-kernel-zero-day)
